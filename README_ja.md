- **リークされたベンチマーク** — コーディングタスクでGPT-5やClaude 4に匹敵または超える性能を示唆

## 期待される機能の詳細

### 1兆パラメータ規模

DeepSeek V4は約**1兆の総パラメータ**を持つと予想されています。しかし、すべてのトークンですべてのパラメータを活性化する密なモデルとは異なり、V4はMixture of Expertsアーキテクチャを使用して推論コストを管理可能に保ちます。

**なぜ重要か：**
- パラメータが多いほど知識容量が大きい
- コードパターン、アルゴリズム、プログラミング概念のより大きな「知識ベース」を格納可能
- 1兆規模のモデルは、小さなモデルにはない創発的能力を示す
- 巨大な規模にもかかわらず、MoEによりトークンあたりの計算量は370億密モデルと同等

### Engramメモリアーキテクチャ

DeepSeek V4で最もエキサイティングなイノベーションは、**Engramメモリアーキテクチャ**です。これは標準的なTransformerのアテンションメカニズムを超える、長文コンテキスト情報検索の革新的なアプローチです。

**Engramメモリとは？**

従来のTransformerは、アテンションがシーケンス長に対して二次的にスケールするため、非常に長いコンテキストの処理に苦労します。Engramメモリは二次的なメモリシステムを導入します：

1. 長距離コンテキストを密な「engram」表現に**圧縮**
2. 効率的な検索のためにこれらの表現を**インデックス化**
3. 検索された情報をモデルのアテンションメカニズムとシームレスに**統合**
4. 生成プロセス全体を通じて**永続化** — 「ワーキングメモリ」のような役割

**開発者にとっての意味：**
- 遠距離の依存関係を見失うことなくコードベース全体を処理
- 100万トークン以上にわたって一貫した理解を維持
- コンテキスト初期の関数シグネチャ、型定義、APIパターンのより良いリコール
- 大規模プロジェクトでのより正確なクロスファイル推論

### 100万トークン以上のコンテキストウィンドウ

DeepSeek V4は**100万トークン**を超えるコンテキストウィンドウをサポートすると予想されています：

- 約75万語のテキスト
- 約25,000ページのドキュメント
- 中規模コードベース全体（100ファイル以上）
- 複数の完全な書籍

### マルチモーダル機能

DeepSeek V4はネイティブにマルチモーダルであると予想され、**テキスト、コード、ビジョン**を統合アーキテクチャで処理します。

**ビジョン機能（予想）：**

## アーキテクチャ

### Mixture of Experts (MoE)

DeepSeek V4はV3で確立されたMoEの伝統を継承しますが、はるかに大きな規模で実現します。

**V4でのMoEの仕組み：**

```
入力トークン
    │
    ▼
┌──────────┐
│  ルーター  │ ── どのエキスパートを活性化するか選択
└──────────┘
    │
    ▼
┌─────────────────────────────────┐
│ エキスパート1 │ エキスパート2 │...│ ── 1T総パラメータ中
│  (アクティブ) │  (アクティブ) │   │    約370億のみアクティブ
└─────────────────────────────────┘
    │
    ▼
┌──────┐
│  出力  │
└──────┘
```

### 効率的な推論

1兆パラメータ規模にもかかわらず、V4は実用的なデプロイメント向けに設計されています：

- トークンあたり**370億のアクティブパラメータ** — 370億密モデルの実行に匹敵
- より高速な生成のための**投機的デコーディング**サポート
- 長文コンテキスト効率のための**KVキャッシュ最適化**
- **量子化対応**アーキテクチャ（FP8/INT4サポート予定）
- **Multi-head Latent Attention (MLA)** — V3から継続、メモリフットプリント削減

## コーディング能力

### リポジトリレベルのコード理解

DeepSeek V4の最も期待される機能の1つは、**真のリポジトリレベルのコード理解**です。現在のモデルは個別のファイルや少数のファイルグループを理解できますが、V4はコードベース全体を一貫したシステムとして理解できると予想されています。

**これが意味すること：**
- プロジェクトの完全な依存関係グラフを理解
- 複数のファイルとパッケージにまたがる型定義を追跡
- アーキテクチャパターン（MVC、マイクロサービス、イベント駆動など）を理解
- 完全なシステムコンテキストでコードスメルとアンチパターンを識別
- すべての下流への影響を考慮したリファクタリングを提案

### マルチファイル推論

リポジトリレベルの理解に基づき、V4は**マルチファイル推論**に優れると予想されています — あるファイルの変更が他のファイルにどう影響するかを理解します。

**能力：**
- クロスファイルの型チェックと推論
- インポート/エクスポートチェーン分析
- モジュール間の副作用追跡
- DBスキーマ ↔ ORMモデル ↔ APIルートの一貫性チェック
- フルスタック推論（フロントエンド ↔ バックエンド ↔ データベース）

### 高度なコード生成

リークされたベンチマークによると、V4は以下を達成する見込みです：

- **HumanEval 約90%** — 標準コーディングベンチマークでほぼ完璧
- **SWE-bench Verified 80%以上** — 人気リポジトリの実際のGitHub Issueを解決
- 大幅な改善：複雑なアルゴリズム実装、システム設計、テスト生成、バグ診断、パフォーマンス最適化

### 5. Engramメモリアーキテクチャとは何ですか？

Engramメモリは、標準的なTransformerアテンションを補助メモリシステムで強化する革新的なアーキテクチャです。非常に長いコンテキスト（100万トークン以上）からの情報を効率的に圧縮、インデックス化、検索できます。

### 6. MoEはどのようにしてV4を1Tパラメータで効率的にしているのですか？

Mixture of Experts（MoE）は各トークンに対してパラメータのサブセットのみを活性化します。V4の場合、1Tパラメータのうち約370億のみが活性化されます。つまり、1兆パラメータモデルの知識容量と約370億モデルの推論速度・コストを両立できます。

### 7. DeepSeek V4は画像を理解できますか？

はい — V4はネイティブにマルチモーダルであり、テキスト、コード、ビジョン入力をサポートすると予想されています。スクリーンショットからコード生成、UI分析、ダイアグラム理解、ビジュアルデバッグなどのユースケースが可能になります。

### 8. V4はどのくらいのコンテキストウィンドウをサポートしますか？

V4はEngramメモリアーキテクチャにより100万トークン以上のコンテキストをサポートすると予想されています。これは単一プロンプトでコードベース全体、ドキュメントセット、または書籍全体を処理するのに十分です。
